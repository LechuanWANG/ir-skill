# 外部证据源与原始资料

只在需要获取公司、交易所、监管、政府或行业机构的网页、PDF、原始披露、政策或行业资料时读取。先确定要验证的假设，再按证据的可靠性、可复核性、时间口径和对决策的影响选择来源与工具；工具开销和获取速度从属于结论质量。需要项目 SQLite 或结构化市场数据时，读取 `tushare-data.md`。

## 来源顺序

按事实的性质而不是按网站机械排序；正式结论、可能改变判断的事项、异常和来源冲突必须回到可追溯的原始材料。不得为减少访问或调用次数而降低关键事实的核验标准。

1. 财报、公告、治理和重大事项的最终事实：在需要正式引用、报告口径/单位/币种/合并范围确认、数据修订、异常值或来源冲突时，使用公司、交易所、巨潮或监管机构的原始 PDF、公告页或直接下载。
2. 政策、宏观定义、行业统计和严格历史可得性：在需要确认统计定义、历史发布版本、政策原文、生效范围、修订或时点时，使用政府、监管、官方统计和行业机构原文。
3. 新闻和市场评论：只用于发现线索；影响研究结论的事实按以上路径核验。
4. 结构化财务、宏观、行情、披露时间线和跨资产比较：读取 `tushare-data.md` 建立可复现的基线；该基线不替代本节所列原始材料的最终事实核验。

## 网页与 PDF

- 搜索入口或官方文件地址时使用搜索工具；已知静态 URL 时直接请求或下载。
- 已知、公开的静态 HTML/PDF 可使用 `scripts/research_collect.py collect --task <task-id> --url <url>`；该工具只把通过类型和内容校验的原件写入 `staging/<task-id>/raw/`，并把安全校验页、错误页和无效 PDF 记录为采集失败。
- 页面依赖 JavaScript、筛选、分页或点击下载时，用浏览器定位真实 PDF、文件 URL 或公开接口，再转回直接下载或脚本处理。
- 只在静态 HTML 需要正文清理或有限抓取时使用 `webclaw`。先检查 `command -v webclaw`；不可用时改用其他来源，安装前取得必要授权。
- 网页无法抽取不等于未披露。抓取文本不替代原始文件核验。
- 财务表格或复杂 PDF 先渲染并查看相关页面，记录页码、表名、报告期、公告日、单位和币种；必要时与官方 HTML、Excel、XBRL 交叉核对。

## 安全与研究完成条件

把 API key、Cookie 和代理凭据保留在环境变量或本地安全配置中，不写入参数文件、报告、缓存、Wiki 或终端输出。结构化数据接口的凭据配置与请求规则见 `tushare-data.md`。

先确认核心假设及其反证已获得与决策风险相称的覆盖，关键数字的来源、口径和时间边界已核验，并记录仍未覆盖的缺口。只有在这些条件满足后，新增资料又合理预期不会改变核心假设、候选排序、行动标签、置信度或触发条件时，才停止扩展；不得仅因时间、调用次数或篇幅目标而停止采集。
