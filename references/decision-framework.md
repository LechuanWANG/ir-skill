# 决策框架：把判断拆开，而非写死

涉及买入、持有、加减仓、卖出或“是否值得”等问题时，使用这份框架避免把不同问题混成一个分数。它是思考检查表，不是固定状态机；根据用户的持有期、约束、数据和问题调整顺序与深度。

## 四个经常需要分开的判断

1. **研究完成度**：现有证据是否足以回答本轮问题？
2. **长期逻辑**：公司、行业或主题在所讨论的持有期内是否有可信的价值来源？
3. **当前赔率与节奏**：价格、估值、事件、流动性和风险是否让今天成为合适的时点？
4. **组合适配**：相对现金、现有持仓和替代标的，风险暴露、集中度和机会成本是否可接受？

用自然语言或用户偏好的标签表达结论即可。不要把研究未完成、公司质量、价格判断和已有持仓动作压缩成一个“买/卖”字，也不要虚构统一枚举以满足模板。

## 条件化比较

对行动性结论，按问题需要比较：

- 基准、上行和下行情景，或至少说明影响结果的关键变量。
- 当前估值/价格与合理范围、盈利路径和风险之间的关系。
- 支持证据、最强反方证据、替代解释和会改变判断的条件。
- 现金和最接近替代标的的机会成本（当用户需要做资源配置时）。

信息不足时使用范围、触发条件和待验证问题，不要制造精确概率、目标价或仓位。用户没有提供风险预算、持仓和交易摩擦时，避免给出貌似确定的配比。

## 反追高与风险

在适用时分别审视：

- **估值风险**：当前价格隐含的增长、利润率或资本回报是否可实现？
- **价格与流动性风险**：近期波动、拥挤、成交和退出成本是否扭曲赔率？
- **叙事风险**：事件、政策、订单或概念是否已经核实，且能否量化传导？

它们不必自动否定公司，也没有通用阈值；解释哪个风险重要、证据是什么以及它改变的是长期逻辑、当前节奏还是组合大小。

## 输出前自查

- 标明 `as_of`、来源口径和关键未知项。
- 通过来源归属、时间边界和条件表达让事实、推断和观点的边界清楚，并保留来源冲突；需要核验或对比时再使用显式提示。
- 给出最重要的反方证据、风险与证伪/复核触发器。
- 让结论与研究深度匹配：归因问题不必输出投资行动，证据不足时可以只给研究计划。
- 用户明确要求复用、维护或更新长期记忆时，才将相关结论与新资料整合到 LLM Wiki；详见 `references/wiki-memory.md`。
