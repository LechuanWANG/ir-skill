# 数据来源与使用边界

先问“哪条假设需要什么证据”，再选来源。数据层支持研究，不替代判断；不要为了使用工具而收集与本轮问题无关的字段。

## 来源优先级

| 问题              | 首选来源                        | 常用补充          | 注意事项                                       |
| --------------- | --------------------------- | ------------- | ------------------------------------------ |
| 财务、重大事项、治理和资本配置 | 公司 IR、交易所、巨潮资讯的 PDF 定期报告与公告 | TuShare、可靠数据库 | 原始披露是最终事实来源；TuShare 只作线索和时间线，区分报告期、公告日和可得日 |
| 价格、估值、成交与复权     | 交易所数据、TuShare、本地市场数据库       | 公司回购/分红披露     | 核对复权、交易日、币种、股本和单位                          |
| 行业供需、产能、价格和政策   | 政府/监管机构、行业协会、公司披露           | 高质量行业研究       | 说明样本、覆盖范围、时点和利益相关方                         |
| 新闻与事件           | 一手公告、监管/政府页面、公司原文           | 至少一条独立可靠报道    | 新闻只提供线索；核实事实和传导后才进入判断                      |
| 宏观和市场背景         | 官方统计、央行、监管机构、交易所            | 权威国际组织或研究机构   | 把宏观变量落到收入、成本、资本成本或风险溢价                     |

## 共同口径

对可能影响结论的数字记录：

```text
value
unit / currency
period_end
published_at
retrieved_at
as_of
source
definition_or_adjustment
```

不要混用报告期、可得时间和价格日期。不同来源的同类数字差异明显时，优先检查定义、合并范围、股本口径、复权、汇率和修订，而不是立即选一个看起来合理的值。

## 数据获取策略

1. 对单股或少量候选，优先获取直接验证核心假设的最小数据包。
2. 对市场扫描或可重复监控，才考虑维护本地基础数据；说明覆盖范围和最后更新时间。
3. 对报告中的财务字段，保留原始 PDF 的页码、表名、报告期、公告日、单位和币种。不要用脚本或 TuShare 复算这些数字来决定报告事实；遇到冲突时回到原始披露解释口径差异。
4. 需要 TuShare 数据时，先按 `references/investment-modes.md` 与 `references/tushare-research.md` 选择模式化数据包；使用 `scripts/tushare_mode_data.py` 获取单股所需的估值、价格、流动性、资金和事件观察，或使用 `scripts/tushare_sync.py` 维护可用的基础表。权限或数据不足时明确说明替代来源。
5. 对财报、公告和公司关键事项，先直连公司、交易所或巨潮的公告页和原始 PDF，再归档原始文件；动态页面用浏览器渲染定位文件或接口。`webclaw` 是静态网页、新闻原文、行业资料和宏观页面的补充或回退工具，不能把无法提取解释成未披露，也不能替代原文。对新闻、行业和宏观信息，先收集、再核实、再解释传导。
6. 用户要求下载、保存或复用资料时，可直接归档小型 CSV/JSON、财报、公告和原始数据到 `raw/<domain>/<subject>/<YYYY-MM-DD>/<内容明确的文件名>`。`domain` 为 `company`、`industry`、`macro` 或 `market`；主题目录使用公司名、行业名或稳定市场名称，不能使用本轮任务名。归档原始资料不要求读取或更新 LLM Wiki。只有用户明确要求维护持久记忆时，才更新对应 `wiki/<domain>/<subject>/<内容页面>.md`、`index.md` 与 `log.md`。大型或多工作表 Excel 留在用户指定位置或工作区 `data/`。

## 本地工具边界

- `scripts/market_data_store.py`：保存和查询结构化市场数据；它不定义研究优先级或投资判断。
- `scripts/tushare_sync.py`：执行基础数据同步；同步范围由本轮问题和可用权限决定。`fina_indicator` 只提供补充趋势线索，不能成为报告的最终财务事实来源。
- `scripts/tushare_mode_data.py`：把长期、中期、短期模式映射为最小 TuShare 数据包，获取数据并记录可用性；它不复算财务报表，也不生成筛选、评级或行动结论。
- `scripts/tushare_gateway.py`：按 Agent 明确选择的 endpoint 和参数获取 TuShare 数据；默认不缓存，不能生成财务核验、筛选、评级或行动结论。
- `webclaw` CLI：为静态公开网页提供可归档的 Markdown/JSON 补充；它不替代原始 PDF、公告页、浏览器渲染或交叉核验，详见 `references/webclaw.md`。
- `scripts/wiki_index.py`：在用户启用或更新 LLM Wiki 后按需检查文档结构；它不验证财务或投资事实。

若数据缺失、过期、无权限或无法交叉核对，直接说明缺口、它对结论的影响以及最有价值的补证路径。
