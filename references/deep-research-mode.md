# 深度研究：四视角交叉质询

在长期投资、复杂单股、重大决策或用户明确要求深度研究时使用。深度研究增加最能检验判断的证据和反方观点，不机械增加角色、轮次或章节。只有用户明确要求行动、且证据足以支撑时，才形成条件化行动判断；补证、等待或暂不判断同样是有效结果。

## 启动与共同证据包

1. 先明确研究问题、持有期、`as_of`、当前已知约束、需要回答的判断，以及哪些结果会改变结论。
2. 先读取本轮直接相关的原始资料和既有结论。只有用户明确要求复用或更新 LLM Wiki 时，才按 `references/wiki-memory.md` 读取相关页面并归档/整合资料；不要因深度研究自动加载历史 Wiki。
3. 建立共同证据包：原始财报和公告、业务分部、资本配置、治理、行业结构、价格/估值、来源冲突和关键缺口。只加入与本轮问题有关的近期事件、宏观、技术面或组合材料。
4. 让所有视角使用同一个 `as_of`、相同的事实边界和来源口径。禁止用决策时点后才公开的信息证明当时的判断。
5. 在独立研究完成前不共享其他视角的结论。先保留独立判断，再开始质询，避免锚定和从众。

运行环境支持 Multi-Agents 时，并行委派四个边界清晰的任务；不支持时，由同一 Agent 按相同输入和输出结构顺序完成，并明确本轮没有独立交叉验证。不要因缺少角色而伪装成委员会结论。

## 四个研究视角

| 视角          | 核心问题                   | 主要关注                       | 必须接受的挑战                     |
| ----------- | ---------------------- | -------------------------- | --------------------------- |
| 生意本质（段永平）   | 这是不是一门能长期创造价值的好生意？     | 客户价值、商业模式、护城河、增长质量、资本配置    | 财务是否验证了生意叙事，竞争会否侵蚀护城河？      |
| 财务与估值（巴菲特）  | 赚的是真钱吗，当前价格留下安全边际吗？    | 利润质量、现金流、资产负债表、再投资回报、估值和情景 | 优秀历史财务能否持续，估值依赖哪些脆弱假设？      |
| 竞争与逆向思考（芒格） | 竞争格局怎样变化，什么会使判断失败？     | 行业结构、竞争对手、周期、替代、激励扭曲和反事实   | 生意和估值假设是否低估竞争、周期或行为偏差？      |
| 风险与治理（李录）   | 什么会造成永久性损失，管理层和治理是否可信？ | 治理、会计、杠杆、流动性、监管、尾部风险和空头论点  | 乐观结论是否忽略红旗，风险是否已被量化并反映在价格中？ |

这些视角是检验问题的透镜，不是四个权威投票席位。任何视角都不能凭角色名称、叙事重复或单一指标决定结论。

## 阶段一：独立研究

每个视角围绕自己的核心问题提交最少而完整的研究意见。对每条会改变结论的关键主张，记录：

```text
claim_id
主张与适用持有期
支持证据与原始来源
反方证据与替代解释
依赖的假设、数据缺口和可观察的证伪条件
对估值、价格条件或研究优先级的影响
初始判断：支持 / 部分支持 / 资料不足 / 反对
```

`claim_id` 与这些字段服务于工作底稿和 Wiki 的可审计性。面向用户的最终报告围绕关键判断自然组织：先说明主张或发现，再交代证据、反方解释、对结论的影响和剩余不确定性；用来源归属、时间边界和条件表达显示证据状态。逐项核验、横向比较或用户要求审计轨迹时，再将字段呈现为显式标签或表格。

生意视角说明价值来源和护城河如何兑现。财务视角以公司和交易所的 PDF 财报、公告等原始披露确认其是否转化为现金流、资本回报和合理估值，不以 TuShare 字段或脚本复算替代财务事实。竞争视角寻找会打破前两者假设的对手、替代、周期和激励。风险视角独立建立最强反方案例，不以“风险提示”代替因果、量级和证据。

## 阶段二：交叉质询

只质询会改变长期逻辑、估值、下行情景或当前行动条件的关键主张。每个关键主张至少接受一个非作者视角的质询；不要求为不重要的背景信息制造辩论。

### Round 1：提出可检验的反驳

质询者记录：

- 被质询的 `claim_id` 和具体结论。
- 它依赖的未证实假设、遗漏的原始证据或错误的时间/口径。
- 最有力的替代解释、反方情景和可能量级。
- 若质询成立，对长期逻辑、估值、价格条件或研究状态的影响。

优先形成以下交叉检查：生意视角质询财务结果是否反映一次性因素；财务视角质询护城河是否真正创造现金；竞争视角质询前两者是否低估对手、周期或替代；风险视角质询所有结论是否忽略治理、会计、杠杆或永久性损失风险。

### Round 2：回应并修订

主张作者只能补充可验证证据、量化影响、用已有事实反驳、缩小主张范围、下调判断或撤回主张。不得用角色权威、重复叙事或新增无来源假设维持原结论。

记录回应后的状态：`支持`、`收窄`、`未解决` 或 `撤回`。仅当出现新的关键证据或仍有未解决的重大矛盾时，才追加下一轮质询。

## 合成与裁决

负责人不拼接四份报告，也不按票数或平均分裁决。负责人按来源可靠性、因果关联、量级、时间边界和对结论的重要性整合研究，明确区分：

1. **共识**：不同视角由独立且可核验的证据支持的结论；共识不等于多数投票。
2. **关键分歧**：仍无法由现有证据解决、且会改变结论的冲突；保留双方论据和下一步验证路径。
3. **盲区**：四个视角都未充分研究、但可能改变判断的问题；将其列为优先研究缺口。
4. **红旗**：欺诈或重大会计嫌疑、治理或诚信红线、偿债/流动性危机、重大合规风险。可靠红旗不能被其他正面观点抵消，必须单独说明其证据、适用范围和对行动条件的影响。

最终结论使用与证据相称的语言：形成条件化观点、继续研究、等待证据、等待价格、回避或暂不判断。只有在用户明确授权且提供必要约束时，才把证券层面的判断延伸到个性化组合操作。

## 深度报告与留痕

深度报告通常包含：

1. 研究问题、持有期、`as_of` 和证据边界。
2. 四视角的核心发现、关键主张及其支持/反方证据。
3. 共识、关键分歧、盲区与交叉质询后的修订。
4. 条件化判断、情景/敏感性、估值或价格条件、主要风险和复核触发器。
5. 数据缺口、未核实来源、过程限制和下一步研究计划。

用户要求沉淀记忆时，将可复用的关键主张、质询、回应、未解决分歧、结论条件和验证触发器整合到 LLM Wiki。原始资料使用 `raw/<domain>/<subject>/<YYYY-MM-DD>/<内容明确的文件名>`；综合结论使用 `wiki/<domain>/<subject>/<内容页面>.md`，再按需更新 `index.md`、`log.md` 与跨领域链接。否则只保留本轮所需的研究底稿与来源。
