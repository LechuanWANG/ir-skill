# 深度研究与独立质询

只在用户明确要求深度研究、独立审阅或交叉质询时使用。先按主 Skill 选择 `long-term.md` 或 `catalyst-trading.md`，再用本文件增加质询；普通单股、候选比较、研究计划和行动判断不得自动升级为多代理流程。

## 启动条件

主 Agent 先准备最小共同证据包：研究对象、决策、持有期、`as_of`、候选和约束、核心假设、原始证据、关键缺口，以及可能改变排序或行动标签的变量。所有审阅者使用相同时间边界，不得引入决策时点之后的信息证明此前判断。

## 可执行的独立审阅

在工具和并发允许时，最多启动三个相互独立的审阅者，主 Agent 保留裁决职责：

1. **商业与财务估值**：商业模式、需求、利润/现金质量、资本回报和当前价格隐含预期。
2. **竞争与替代解释**：竞争者反应、供需周期、扩产降价、技术替代，以及对乐观因果链的攻击。
3. **治理与尾部风险**：会计、杠杆、流动性、治理、监管、诚信和永久性损失。

并发不足时使用一至两个审阅者合并覆盖角色；完全不能调用独立审阅者时，由主 Agent 完成明确标注的对立案例检查，并说明这不是独立复核。不得伪称使用了多个 Agent。

审阅者在独立结论完成前不看其他审阅者的判断。每位只提交：

- 当前立场及两至四项关键判断；
- 最强原始证据；
- 最有力的反方证据或替代解释；
- 最可能被忽略的变量；
- 可观察的证伪条件和当前置信度。

候选比较时同时给出自己的排序或行动倾向。不要要求固定长报告或逐项复述共同证据。

## 交叉质询

主 Agent 只转交可能改变核心假设、估值、排序、主要风险或行动标签的冲突。质询应指出具体主张、未证实假设、证据或时间口径问题，以及质询成立时会如何改变决策。

原审阅者通过补充证据、量化影响、收窄范围、降低置信度、修改排序或撤回主张回应。将结果标为 `支持`、`收窄`、`未解决` 或 `撤回`。当新一轮不再改变决策时停止，不为展示讨论而延长流程。

## 主 Agent 裁决

不要按票数或平均分决定。综合证据强度、因果关系、影响量级、赔率和错误损失，明确：

- 独立证据支持的共识；
- 仍可能改变决策的关键分歧，以及当前证据更支持哪一方；
- 尚未覆盖的盲区、治理/流动性红旗和已被撤回的观点；
- 当前排序或单股判断、主要行动标签、现金/替代项比较；
- 两至四个决策变量、最可能推翻结论的证据、触发条件和复核时间。

最终报告只展示真正改变判断的质询、回应和修订，不必输出完整对话，也不得披露模型私有推理过程。
