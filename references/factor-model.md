# 因子研究：明确、可解释、可替换

只有当用户明确要求多因子、量化初筛、动量或系统化比较时才使用本文件。因子模型帮助缩小研究范围或检验某个假设；它不自动给出长期评级、买卖信号或仓位。

## 先写研究设计

在查看结果前说明：

- 股票池、市场、`as_of`、再平衡频率和持有期。
- 要解决的经济问题，例如低估值修复、质量持续性、增长、趋势、风险控制或行业相对比较。
- 可获得的数据、复权和财务可得时间的处理方式。
- 使用哪些因子、为什么使用、可能偏向或遗漏哪些公司。
- 如何把结果用于后续研究，而不是直接当作投资结论。

避免把所有风格相加为一个“万能分数”。权重、门槛、行业中性化、流动性要求和风险惩罚应由本轮问题决定并披露；若使用基准模型，保留它与自定义方案的差异。

## 可选维度

| 维度 | 常见观察 | 需要警惕 |
|---|---|---|
| 价值 | 盈利、账面、现金流、股息相对估值 | 周期顶部、会计口径、价值陷阱 |
| 质量 | ROIC/ROE、利润率、现金转换、资产负债表 | 行业资本结构和一次性利润 |
| 增长 | 收入/利润/现金流增长、份额、再投资 | 基数效应、不可持续补贴或并购 |
| 趋势 | 相对收益、趋势稳定性、修正后动量 | 追高、拥挤和反转风险 |
| 风险与可交易性 | 波动、回撤、流动性、集中度 | 数据缺失、新股、停牌和幸存者偏差 |

根据行业和问题挑选少量有经济含义的维度。不要把缺失值默认当作中性好消息；记录数据缺口及其对排名的影响。

## 输出与复核

量化输出至少说明股票池、日期、数据来源、计算口径、筛选条件、参数、结果和局限。把结果称为“研究线索”或“候选队列”，随后用 `references/research-screening.md`、基本面和原始披露进行独立检验。

若用户没有明确量化请求，优先从商业问题、行业结构和用户提供的候选出发。不要因为脚本或历史模型存在就自动运行筛选。
